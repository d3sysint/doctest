RESTful Web API v2¶

Introduction¶

The Web API enables extensive automation of major task such as sample uploading, report downloading, searching and more. The following is a documentation of the API. All API calls are implemented as POST requests.

The API endpoint is: https://jbxcloud.joesecurity.org/api/

The POST requests must be encoded as multipart/form-data.

All Boolean values (true/false) specified in the routes below need to be encoded using 1 and 0.

Python Scripts¶

Joe Security provides a convenient implementation of the API for Python: jbxapi.py. It allows to easily integrate Joe Sandbox in your environment

Some more useful scripts are available on our Github page.

API Key¶

To generate an API key, go to the “User settings” section in the User menu and select the “API Key” tab.

Responses¶

On success, the API responds with a HTTP status 200 and a JSON object containing the response under the data key:

{
    "data": {
        "submission_id": "1"
    }
}

Pagination¶

Some endpoints support pagination. If pagination is enabled, there is an extra key pagination in the response object. The JSON object looks as follows:

{
    "data": [{
        "webid": "10001"
    },
        ...
    ],
    "pagination": {
        "next": "1234"
    }
}

You can page through the results by sending an additional parameter pagination_next=1234.

It is possible that fewer than the maximum amount of objects are returned per request (or even no objects at all).

You reached the end of the list once there is no next key present.

Note

Do not cache the pagination tokens, they can become invalid at any time. Treat the tokens as opaque objects, we reserve the right to change them without notice in the future.

on error¶

On error, the JSON object contains errors with a list of error objects containing a machine-readable code and a human-readable message like this:

{
    "errors": [{
        "code": 2,
        "message": "Parameter apikey is required."
    }]
}

A list of error codes and associated HTTP status can be found here: Error Codes.

Discussion¶

If you set one of the filters, the route returns only webids of analyses that match these.

If several of the other fields are set, all of them must match.

The search is case-insensitive.

For some fields, only exact matches are considered, for others also substring matches are valid (see the description column above). E.g. if a URL is “https://www.google.com/my-page”, the url search string “google.com” will match.

before-date and after-date do not include the given date. On Cloud, CET/CEST time zone is applied.

Discussion¶

The route returns analyses that match the given search parameters.

You must either set q or one or several of the other search parameters.

If q is set, the server searches the following fields for it: md5, sha1, sha256, filename, threat name, URL, tags, comments

If several of the other fields are set, all of them must match.

The search is case-insensitive.

For some fields, only exact matches are considered, for others also substring matches are valid (see the description column above). E.g. if a URL is “https://www.google.com/my-page”, the url search string “google.com” will match.

before-date and after-date do not include the given date. On Cloud, CET/CEST time zone is applied.

Note

In case too many analyses match the conditions, the route returns an error object.

In this case, either refine your conditions, or use the /v2/analysis/list route that supports pagination.

HTTP Status¶

The API responds with an appropriate HTTP status. We currently use 200 for good responses, 4xx for errors and 503 to show that the server is offline or under maintenance.

Error Codes¶

When the API encounters an error, a JSON error object is passed to the client as described above. Some errors have a specific error code so the client can react accordingly.

Typically an appropriate HTTP status code is supplied.

Error Code

Typical HTTP status

Description

1

400

An unspecified error happened.

2

400

An API parameter is missing.

3

400, 404

An API parameter is invalid.

4

403

The API key is invalid.

5

503

Joe Sandbox is offline.

6

500

An internal server error occurred.

7

403

The user does not have the required permissions.
